## Проект

Небольшая браузерная игра match-3 на Phaser 3 + TypeScript, собранная на Vite.

## Цель проекта

Сделать базовую playable-версию match-3 без серверной части: с игровым полем, базовой логикой совпадений, подсчётом очков и возможностью перезапуска.

## Стартовое состояние проекта

- Проект находится на этапе стартового прототипа.
- Приложение уже запускает Phaser внутри страницы Vite.
- В `src/main.ts` собрана минимальная оболочка страницы с заголовком, строкой технологий и крупным контейнером под игру.
- В `src/game/config.ts` задан базовый конфиг Phaser: размер 960x540, `FIT`, `CENTER_BOTH`, подключены сцены `BootScene` и `MainMenuScene`.
- `BootScene` сразу переводит приложение на основную игровую сцену.
- Добавлена базовая `GameScene`: сейчас она показывает стартовое поле, собранное из модели в памяти.
- Добавлена минимальная модель поля и стартовая генерация без готовых матчей.
- Добавлены выбор фишки и swap между соседними клетками с простой анимацией.
- Добавлены поиск совпадений 3+ по горизонтали и вертикали, проверка результата swap и откат невалидного хода.
- Добавлены удаление совпавших фишек и базовое начисление очков.
- Добавлено падение фишек вниз после удаления совпадений.
- Добавлено дозаполнение поля новыми фишками сверху после падения.
- Добавлена обработка каскадов после дозаполнения до полной стабилизации поля.
- Пользовательский ввод блокируется на время swap, анимаций и автоматической обработки хода.
- На сцене отображается текущий счёт и добавлена кнопка рестарта.
- Рестарт выполняется через перезапуск сцены с полным пересозданием поля и сбросом счёта.
- Добавлено управление через drag-and-drop: фишку можно перетаскивать в сторону соседней клетки для обмена.
- Добавлена возможность менять число строк и столбцов поля во время игры с автоматическим сбросом сцены.
- Добавлена возможность менять количество активных цветов фишек во время игры в диапазоне от 2 до 6 с автоматическим сбросом сцены.
- Игровая сцена переведена на адаптивный layout: на десктопе элементы управления остаются слева от доски, а на мобильных разрешениях переносятся под поле.
- DOM-оболочка страницы перестроена: заголовок и строка технологий теперь находятся внутри общего контейнера с игрой; на десктопе `game-root` занимает оставшуюся высоту экрана, а на мобильных разрешениях сохраняется вертикальный порядок "заголовок -> поле -> элементы управления".
- На десктопе заголовок и строка технологий перенесены в canvas слева над HUD, а DOM-заголовок скрыт; на мобильных DOM-заголовок остаётся сверху как отдельный блок над игрой.
- Для desktop layout убрано ограничение роста доски базовым `CELL_SIZE`: размер ячейки теперь рассчитывается от доступной ширины и высоты, чтобы маленькие поля тоже занимали максимум свободного места.
- При изменении размеров окна Phaser-сцена теперь автоматически пересобирается с сохранением текущего состояния поля, счёта и настроек, поэтому layout подстраивается без ручной перезагрузки страницы.
- Конфиг Phaser переведён на `RESIZE`, чтобы canvas совпадал по реальному размеру с контейнером и не размывался из-за дополнительного CSS-масштабирования.
- Добавлены звуковые эффекты для swap и match, а также зацикленная фоновая музыка во время игры.
- Фоновая музыка сделана тише по умолчанию и вынесена на отдельный ползунок громкости без влияния на эффекты.

## Текущая структура проекта

- `src/main.ts` — входная точка приложения, разметка оболочки страницы, создание экземпляра Phaser.
- `src/style.css` — стили общей страницы, единого контейнера с заголовком и игрой, без жёсткого `aspect-ratio` для canvas-области и с отдельными desktop/mobile правилами по высоте.
- `src/game/config.ts` — базовая конфигурация Phaser.
- `src/game/constants.ts` — стартовые значения размеров поля, размеры ячеек, типы фишек и ограничения для числа активных цветов.
- `src/game/boardModel.ts` — минимальная модель поля, генерация стартового состояния, удаление матчей, падение и дозаполнение фишек для текущих размеров доски и активного набора цветов.
- `src/game/game.ts` — фабрика создания `Phaser.Game`.
- `src/game/matchFinder.ts` — поиск совпадений 3+ по горизонтали и вертикали.
- `src/game/types.ts` — базовые типы координат, типов фишек, состояния поля, матч-групп и настроек доски.
- `src/game/scenes/BootScene.ts` — стартовая сцена, предзагрузка аудио-ассетов и переход в игру.
- `src/game/scenes/GameScene.ts` — игровая сцена, отрисовка поля, адаптивный layout с desktop-заголовком внутри canvas, раздельным desktop/mobile размещением HUD, реакцией на `resize` без перезагрузки страницы, обработкой выбора, swap, drag-and-drop, матчей, счёта, рестарта, смены размеров доски, количества цветов и воспроизведения звука.
- `src/game/scenes/MainMenuScene.ts` — прежний экран-заглушка, сейчас больше не используется в конфиге сцен.
- `public/` и `src/assets/` — статические ассеты проекта.
- `package.json` — стандартные скрипты `dev`, `build`, `preview`, `lint`, `format`, `format:check`.

## Важные замечания

- Настройки ESLint и Prettier были вручную изменены после первоначальной генерации проекта, поэтому их нельзя считать стандартными шаблонными.
- При дальнейшей работе не менять конфиги без отдельной необходимости и явного понимания причины.
- Текущее содержимое `eslint.config.js` выглядит как ранее вручную адаптированный конфиг и не похоже на исходный минимальный конфиг Vite для этого проекта.
- Игра будет реализовываться пошагово, без попытки собрать всю механику за один крупный этап.
- Swap считается валидным только в том случае, если после него на поле появляется хотя бы один матч.
- Стартовое поле должно генерироваться без готовых матчей.
- На старте используем минимальную архитектуру без лишнего дробления на модули: сначала несколько простых файлов с понятной ответственностью, без преждевременного усложнения.
- Обработка каскадов добавлена прямо в `GameScene` через простой цикл стабилизации поля, без введения отдельной state machine.
- Для рестарта выбран перезапуск сцены как самый короткий и надёжный способ сбросить визуальное и игровое состояние синхронно.
- Drag-and-drop встроен в существующий `GameScene` и использует уже готовую анимацию swap, чтобы не дублировать логику обмена.
- Для внешнего layout сохранён минимальный подход: без отдельного UI-фреймворка и без дополнительной структуры поверх текущей страницы.
- Размеры доски переведены из жёстко фиксированных констант в текущее состояние сцены; константы теперь используются как стартовые значения по умолчанию.
- Количество активных цветов тоже хранится в состоянии `GameScene`; по умолчанию активны все 6 цветов из `gems.png`, а для поддержки режима с 2 цветами генерация и дозаполнение переведены с жадного выбора на backtracking без стартовых матчей.
- Для адаптивности размер клетки больше не фиксирован строго: `GameScene` вычисляет его из доступной ширины и высоты canvas отдельно для desktop и mobile layout; в мобильном режиме ширина доски старается занять всю доступную ширину контейнера.
- Для борьбы с размытием canvas используется режим `RESIZE`: канвас получает реальный размер контейнера вместо последующего растягивания через `FIT`.
- Аудио загружается в `BootScene`, а логика воспроизведения остаётся в `GameScene`, чтобы эффекты были привязаны к игровым событиям без отдельного аудио-менеджера.
- Для замены процедурных ромбов используется `spritesheet` `assets/sprite/gems.png`: существующие типы фишек сопоставлены с цветными кадрами без расширения набора игровых типов.
- Фоновая музыка запускается только когда аудио не заблокировано браузером, и не дублируется при `scene.restart()`.
- Отдельный ползунок меняет только громкость фоновой музыки; громкость эффектов хранится отдельно и не зависит от него.
- Кнопка mute отключает только музыку, не трогает эффекты и при повторном нажатии возвращает музыку на сохранённый уровень ползунка.
- Повышение высоты `match` работает только внутри одного каскадного цикла и сбрасывается после его завершения, чтобы следующий ручной матч снова начинался с базового тона.
- На уровне Phaser для `SoundManager` отключена пауза по blur (`pauseOnBlur = false`), но браузерные ограничения фонового аудио всё равно могут зависеть от конкретной платформы.
- Для GitHub Pages используется Vite `base` через переменную окружения `VITE_BASE_PATH`: локально это `/`, а в GitHub Actions автоматически подставляется путь вида `/<repo-name>/`.
- Публикация на GitHub Pages настроена через workflow `.github/workflows/deploy-pages.yml` и официальный сценарий GitHub Pages artifact deploy.

- `src/game/scenes/GameScene.ts` — основная игровая сцена для первого рабочего прототипа поля.
- `src/game/constants.ts` — константы поля, размеров и базовых параметров игры.
- `src/game/types.ts` — простые типы для координат, фишек и состояния поля.

- `src/game/config.ts` — подключение новой игровой сцены, когда начнётся реализация.
