## Проект

Небольшая браузерная игра match-3 на Phaser 3 + TypeScript, собранная на Vite.

## Цель проекта

Сделать базовую playable-версию match-3 без серверной части: с игровым полем, базовой логикой совпадений, подсчётом очков и возможностью перезапуска.

## Стартовое состояние проекта

- Проект находится на этапе стартового прототипа.
- Приложение уже запускает Phaser внутри страницы Vite.
- В `src/main.ts` собрана оболочка страницы с текстовым hero-блоком и контейнером под игру.
- В `src/game/config.ts` задан базовый конфиг Phaser: размер 960x540, `FIT`, `CENTER_BOTH`, подключены сцены `BootScene` и `MainMenuScene`.
- `BootScene` сразу переводит приложение на `MainMenuScene`.
- `MainMenuScene` сейчас играет роль стартового экрана-заглушки: рисует фон, декоративную сетку фишек и текст, но игровой логики match-3 ещё нет.
- Структуры для поля, фишек, матчей, падения и подсчёта очков пока нет.

## Текущая структура проекта

- `src/main.ts` — входная точка приложения, разметка оболочки страницы, создание экземпляра Phaser.
- `src/style.css` — стили общей страницы и контейнера с игрой.
- `src/game/config.ts` — базовая конфигурация Phaser.
- `src/game/game.ts` — фабрика создания `Phaser.Game`.
- `src/game/scenes/BootScene.ts` — стартовая сцена-переход.
- `src/game/scenes/MainMenuScene.ts` — текущий экран-заглушка вместо реальной игровой сцены.
- `public/` и `src/assets/` — статические ассеты проекта.
- `package.json` — стандартные скрипты `dev`, `build`, `preview`, `lint`, `format`, `format:check`.

## Уже выполненные шаги

- Создан базовый проект на Vite + TypeScript.
- Подключён Phaser 3.
- Настроен запуск игры внутри DOM-контейнера.
- Добавлены базовые сцены `BootScene` и `MainMenuScene`.
- Собран стартовый визуальный экран-прототип вместо пустого шаблона.
- Подготовлена базовая структура папки `src/game/` для дальнейшего развития игровых сцен.

## Важные замечания

- Настройки ESLint и Prettier были вручную изменены после первоначальной генерации проекта, поэтому их нельзя считать стандартными шаблонными.
- При дальнейшей работе не менять конфиги без отдельной необходимости и явного понимания причины.
- Текущее содержимое `eslint.config.js` выглядит как ранее вручную адаптированный конфиг и не похоже на исходный минимальный конфиг Vite для этого проекта.
- Игра будет реализовываться пошагово, без попытки собрать всю механику за один крупный этап.
- Swap считается валидным только в том случае, если после него на поле появляется хотя бы один матч.
- Стартовое поле должно генерироваться без готовых матчей.
- На старте используем минимальную архитектуру без лишнего дробления на модули: сначала несколько простых файлов с понятной ответственностью, без преждевременного усложнения.

## Согласованный пошаговый план реализации

1. Добавить отдельную игровую сцену вместо текущего стартового экрана-заглушки.
2. Зафиксировать базовые константы и типы: поле 8x8, размеры ячеек, набор типов фишек, базовые структуры данных.
3. Реализовать минимальную модель поля и генерацию стартового состояния без готовых матчей.
4. Отрисовать поле и фишки в `Phaser`, связав визуальное представление с моделью.
5. Добавить выбор фишек и swap только между соседними клетками.
6. Реализовать проверку swap: если матч не создан, обмен откатывается как невалидный.
7. Реализовать поиск совпадений 3+ по горизонтали и вертикали.
8. Добавить удаление совпавших фишек и начисление очков.
9. Реализовать падение фишек вниз в освободившиеся ячейки.
10. Реализовать дозаполнение поля новыми фишками сверху.
11. Добавить обработку каскадов: повторение цикла матчей, удаления, падения и дозаполнения до стабилизации поля.
12. Добавить HUD: отображение счёта и кнопку рестарта.
13. Провести ручную проверку полного базового цикла игры.

## Предполагаемые стартовые файлы для первого этапа

- `src/game/scenes/GameScene.ts` — основная игровая сцена для первого рабочего прототипа поля.
- `src/game/constants.ts` — константы поля, размеров и базовых параметров игры.
- `src/game/types.ts` — простые типы для координат, фишек и состояния поля.
- `src/game/boardModel.ts` — минимальная модель поля, генерация стартового состояния и базовые операции над сеткой.
- `src/game/matchFinder.ts` — поиск совпадений 3+ без лишнего усложнения.
- `src/game/ui/Hud.ts` или временно логика HUD внутри `GameScene` — в зависимости от объёма первого этапа.
- `src/game/config.ts` — подключение новой игровой сцены, когда начнётся реализация.

На первом этапе лучше начать с минимального набора:

- `src/game/scenes/GameScene.ts`
- `src/game/constants.ts`
- `src/game/types.ts`
- `src/game/boardModel.ts`
- `src/game/matchFinder.ts`
